## With clear button

<Playground>
  {() => {
    const [value, setValue] = React.useState('Clear me');
    return (
      <Box width={300}>
        <Stack>
          <TextField
            label="Allows clearing the input"
            value={value}
            onChange={(e) => setValue(e.target.value)}
            onClear={() => setValue('')}
          />
          <TextField
            label="Disabled"
            value={value}
            onChange={(e) => setValue(e.target.value)}
            onClear={() => setValue('')}
            disabled
          />
        </Stack>
      </Box>
    );
  }}
</Playground>

#### Be aware

If you provide `defaultValue`, but not `value` prop, the clear button won't be shown, as it only checks for `value` prop.
To fix this, `TextField` component will need to have an internal state, making the `<input />`
always controlled. This creattes additional overhead and code to cover such rare case.

<Playground>
  <Box width={300}>
    <TextField label="Allows clearing the input" onClear={() => {}} inputProps={{ defaultValue: 'Clear me' }} />
  </Box>
</Playground>
